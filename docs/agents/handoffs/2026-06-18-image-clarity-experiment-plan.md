# 拍立得提取清晰度实验方案

日期：2026-06-18

目的：整理关于提高清晰度的讨论，作为交给其他项目 agent 的实验说明。实验目标是在严格保持原始人像、文字、签名和图像内容不变的前提下，提高提取后拍立得的主观清晰度。

## 硬性约束

- 不使用生成式图像模型。
- 不使用人脸修复、人脸重绘、人像美化、人脸增强，或任何可能改变身份、表情、皮肤纹理、眼睛、嘴、发际线、文字、签名、印刷细节的模型。
- 不使用任何会幻觉、脑补、语义改变图像内容的算法。
- 允许的增强应只影响对比度、噪声、插值、边缘强调或几何映射，不应创造新内容。
- 如果实验未来迁回 Chekinana，必须保持 RunPod 启动、auth/token、任务队列/中断、mini/wide/auto 尺寸、结果路由等生产合同兼容。

## 方案 A：保守非生成式清晰度增强

状态：建议作为 V1 实验。

目标：提高观感清晰度，但不改变内容。

候选操作：

- 透视矫正后做轻量 unsharp mask。
- 只在亮度通道做低强度局部对比度增强，例如弱 CLAHE。
- 重新调低降噪强度，避免抹掉脸部边缘、文字和签名笔画。
- 对比关闭降噪、当前降噪、弱降噪加锐化、仅锐化等版本。

建议实验变体：

- `A0`: 当前生产基线。
- `A1`: 关闭降噪，不锐化。
- `A2`: 当前降噪后加轻量 unsharp mask。
- `A3`: 弱降噪后加轻量 unsharp mask。
- `A4`: 亮度通道弱 CLAHE 后加轻量 unsharp mask。

风险：

- 高反差边缘出现 halo。
- 纸面颗粒、JPEG 块、噪声被增强。
- 参数过强时，人脸和文字边缘会显得硬。

验收标准：

- 人像身份和表情不变。
- 不产生新的人脸纹理。
- 不改变文字、签名、印刷细节。
- 内部图像边界没有明显 halo。
- 在真实手机照片上，主观清晰度优于当前基线。

## 方案 B：根据四边形原始像素尺度调整输出尺寸

状态：保留为 V2 候选，不建议进入 V1。

这里的“检测到的四边形实际像素尺寸”指输入原图坐标系中四条边的像素长度。四边形不需要水平竖直，因为测量的是沿拍立得自身边缘方向的距离：

```text
上边 = distance(左上, 右上)
右边 = distance(右上, 右下)
下边 = distance(右下, 左下)
左边 = distance(左下, 左上)

估算宽度 = (上边 + 下边) / 2
估算高度 = (左边 + 右边) / 2
```

潜在收益：

- 如果原图中拍立得区域只有约 `700x1100` 像素，强行输出 `1600x2544` 不会产生真实细节，反而可能显得糊。
- 如果原图中拍立得区域远大于 `1600x2544`，固定输出会损失已有细节。
- 自适应输出能避免过度放大或过度缩小。

风险：

- 当前生产输出尺寸稳定，改变尺寸会影响下游假设和用户预期。
- 同一批次可能出现不同尺寸结果。
- 若未来采用，应作为质量模式或内部有上限的缩放策略，不应静默替换当前合同。

建议实验：

- 保持生产固定尺寸不变。
- 离线生成对照输出：
  - 当前固定尺寸。
  - 接近源四边形像素尺度的输出。
  - 有上限的输出，例如限制最大放大倍数。
  - 插值对比：`INTER_CUBIC` vs `INTER_LANCZOS4`。

## 方案 RL：Richardson-Lucy / 去卷积去模糊

状态：只做实验，默认关闭。

Richardson-Lucy 是非生成式去卷积算法。它适合模糊核相对稳定的情况，例如轻微失焦或简单手抖。

潜在收益：

- 当模糊核合理时，比单纯锐化更有依据。
- 不以重绘人脸为目标。
- 可能恢复轻微模糊照片中的边缘对比度。

风险：

- 需要估计或选择 PSF / 模糊核。
- 手机拍摄模糊常常是空间变化的，全图一个核可能不成立。
- 容易放大噪声、JPEG 压缩块、纸面颗粒和振铃。
- 迭代过多会产生假边缘；虽然不是生成式，但观感仍可能不真实。

建议实验：

- 只在亮度通道应用，再和颜色通道合成。
- 迭代次数严格限制，例如 `5`、`8`、`10`。
- 分别测试小 Gaussian PSF 和简单 motion PSF。
- 可测试轻微前置或后置降噪，但避免强降噪。

验收标准：

- 眼睛、嘴、文字、高反差边界周围不能出现明显振铃。
- 必须在多张真实样张上明显优于简单 unsharp mask，才考虑继续。
- 如果只是变硬、变脏或噪声更明显，应拒绝。

## 方案 SR：EDSR / RCAN 超分

状态：不进入默认流程；仅在用户明确接受预测纹理风险时，作为离线实验。

EDSR、RCAN 是深度学习超分模型。它们通常不是人脸修复 GAN，但仍会根据训练数据预测高频细节。

潜在收益：

- 对低分辨率裁切图可能有较强观感提升。
- 边缘和纹理可能优于传统插值。

当前约束下的风险：

- 可能改变皮肤纹理、头发、衣服纹理、印刷纹理和签名笔画。
- 可能增强纸面颗粒和压缩噪声。
- 即使不是人脸修复，也不能视为严格内容保持。
- RunPod 推理成本和批量处理耗时可能明显增加。

当前建议：

- 不作为 V1，也不作为默认生产方案。
- 禁止 GFPGAN、CodeFormer 或任何人脸修复模型。
- 如果测试 EDSR/RCAN，只能作为离线实验产物，并明确标记为“不满足严格人像不变要求的生产安全方案”。

## 方案 Geometry：纸框四顶点 + 内部图像区域四顶点

状态：有潜在正收益，建议作为 V2 实验。

问题：同时使用拍立得外框四点和内部图像区域四点，是否能改善透视矫正和观感清晰度？

潜在收益：

- 当前生产主要依赖外框四点矫正整张拍立得。
- `cheki` 已经证明可以在矫正后纸框中检测内部图像区域。
- 如果内部图像区域顶点是独立检测/拟合得到的，它提供了新的几何信息。
- 更准确的内部图像几何可能减少拉伸、偏斜，从而让内容看起来更自然、更清晰。

重要区别：

- 如果内部图像区域四点只是由纸框四点按固定比例推算，则几乎没有新增信息，收益很小。
- 可能收益来自“独立检测或拟合内部图像边界”。

实验选项：

1. 纸框基线：
   - 只用外框四点。
   - 对应当前生产风格。

2. 两阶段内部图像区域矫正：
   - 外框四点矫正整张拍立得。
   - 在矫正后的纸框中检测内部图像区域四点。
   - 单独把内部图像区域矫正为标准矩形用于评估。
   - 这接近 `cheki\run_pipeline_check.py` 和 `cheki\extract_polaroid_batch.py` 的做法。

3. 八点 homography 拟合：
   - 同时使用外框四点和内部图像区域四点，估计一个最佳折中的 homography。
   - 如果两组点都可靠，可能比只用外框更稳。
   - 但当纸张弯曲或两组检测冲突时，一个 homography 不能同时完美满足八个点。

4. 分区域组合：
   - 外框区域使用外框 homography。
   - 内部图像区域使用内部图像 homography。
   - 再合成到最终拍立得模板。
   - 可能改善内部图像，但图像区域边界处可能出现接缝或不连续。

推荐实验顺序：

1. 在多张接近当前用户场景的照片上复现 `cheki` 两阶段输出。
2. 对比当前生产输出和内部图像区域单独矫正输出。
3. 判断内部图像中的人脸、文字、直线是否更自然、更少拉伸。
4. 再考虑八点 homography 或分区域组合。

风险：

- 内部图像区域检测可能失败，或误检到照片内容内部。
- 纸张弯曲时，全局 homography 可能不足。
- 分区域组合可能在照片/白边交界处产生可见不连续。

验收标准：

- 不裁掉有意义内容。
- 内部图像不能被明显拉伸或压缩。
- mini/wide 模板几何仍然正确。
- 内部区域检测不确定时，必须 fallback 到当前固定 `IMAGE_AREA_VERTICES` 行为。

## 已丢弃 / 禁止的方向

已丢弃：

- 大范围角点/边缘精修作为生产默认改动。该方向容易引入裁切和检测回归，除非后续有强证据。

禁止：

- 人脸修复。
- 生成式超分。
- 任何会重绘或语义改变人脸、文字的模型。
- 任何人像美化或肖像增强。

## 建议实验矩阵

至少使用以下样张：

- 3 张清晰手机照片。
- 3 张典型模糊手机照片。
- 2 张低光/高噪声照片。
- 2 张含手写文字或签名的照片。
- 至少一个 `mini` 和一个 `wide` 场景。

每个样张保存：

- 输入图。
- 纸框 mask overlay。
- 纸框基线矫正输出。
- 内部图像区域 mask overlay，如果使用。
- 内部图像区域矫正输出，如果使用。
- 白平衡后输出。
- 每个清晰度增强变体。
- 包含参数和耗时的 JSON report。

建议变体顺序：

1. 当前生产基线。
2. 关闭降噪。
3. 弱降噪。
4. 弱降噪 + 轻量 unsharp mask。
5. 亮度通道 CLAHE + 轻量 unsharp mask。
6. Richardson-Lucy mild，默认关闭。
7. 两阶段内部图像区域几何矫正。
8. 两阶段几何矫正 + 最佳保守清晰度增强。

不要把 EDSR/RCAN 放入主实验矩阵，除非明确标记为离线实验且不满足当前严格生产约束。

## 评估标准

主观检查：

- 人脸是否仍是同一个人、同一个表情。
- 眼睛、嘴、发际线、文字、签名是否没有变化。
- 在手机正常观看尺寸下是否更易查看。
- halo、振铃、JPEG 块、纸面颗粒是否可接受。

几何检查：

- 照片内容是否更少偏斜。
- 直线边缘是否更自然。
- 是否裁掉有意义内容。
- mini/wide 模板几何是否保持正确。

运行检查：

- 单张拍立得耗时。
- GPU/CPU 内存使用。
- 批量处理时是否仍可接受。
- 失败 fallback 是否能返回当前风格的可用输出。

## 推荐下一步

先在实验项目中做两个离线脚本，不直接改生产代码：

1. `clarity_variants.py`
   - 输入：已矫正的拍立得 PNG。
   - 输出：A1-A4 保守增强变体和 RL mild 变体。

2. `geometry_variants.py`
   - 输入：原始手机照片和检测到的纸框 mask。
   - 输出：纸框基线、两阶段内部图像区域矫正、可选八点拟合及报告。

视觉比较后，只选择一个很窄的生产候选：

- 弱降噪 + 轻量 unsharp mask；或
- 降低/关闭当前降噪 + 轻量锐化；或
- 如果证据充分，采用带 fallback 的内部图像区域几何改进。
